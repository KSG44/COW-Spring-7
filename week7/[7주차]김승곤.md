## 1. Spring Boot
기존 자바 기반의 오픈 소스 프레임워크인 Spring의 복잡한 설정을 자동화하여 개발 생산성을 극대화한 프레임워크이다.\
기존 스프링은 설정이 복잡하고 초기 환경 구축에 시간이 많이 소요되었는데, 스프링 부트는 이를 자동화한 것이다.

*   **주요 특징**
    *   **자동 설정(Auto Configuration)**: 공통적으로 필요한 프로젝트 설정을 프레임워크가 자동으로 처리할 수 있다. \예를 들면 특정 라이브러리가 클래스 패스에 있으면 자동으로 필요한 설정을 적용하는 방식이다.
    *   **단독 실행**: 내장 톰캣(Tomcat)을 사용하여 별도의 서버 설치 없이 실행 할 수 있다.
    *   **의존성 관리**: 여러 라이브러리를 하나하나 설정할 필요 없이 묶음 단위로 관리할 수 있다. 라이브러리 버전 호환성 고민 x

|         | Spring                       | Spring Boot             |
|:--------|:-----------------------------|:------------------------|
| **설정**  | XML이나 자바 코드로 복잡한 설정 필요       | 자동 설정을 통해 설정 최소화        |
| **서버**  | 별도의 WAS(Tomcat 등) 설치 및 설정 필요 | 내장 서버를 사용하여 바로 실행 가능    |
| **의존성** | 각 라이브러리 버전을 직접 맞추어야 함        | Starter를 통해 버전 조합 자동 관리 |

## 2. Spring DI (Dependency Injection)
객체 간의 의존 관계를 직접 생성하지 않고, **Spring 컨테이너가 외부에서 주입**해주는 디자인 패턴이다.\
제어의 역전이라고 개발자가 객체의 생명주기를 관리하는 것이 아닌 이 스프링이 그 권한을 가지고 객체를 생성하고 관리하는 것이다.

### 의존성 - dependency
한 객체가 다른 객체를 필요로 하는 관계. \ 
일반적인 방식으로는 A라는 클래스 안에 B라는 객체를 직접 생성하는데 의존성 주입을 사용하면 A라는 클래스는 B라는 객체를 사용하겠다는 선언만 할뿐 직접 생성하지 않는다. 실제 객체는 외부에서 넣어준다. 이러면 결합도를 낮출 수 있다.\
#### 장점
왜 사용하나? 
1. 결합도 감소 : 객체 간의 결합도를 낮추면 코드 수정이 쉬워진다.
2. 재사용성 및 유지보수성 향상 : 인터페이스를 활용해 구현체를 쉽게 바꿀 수 있다.
3. 테스트 용이성 : 가짜 객체를 주입하여 단위 테스트가 가능하다.

### 스프링 DI 방식
#### 1. 생성자 주입
생성자를 통해 의존성을 주입받는 방식. \
가장 권장되는 방식으로, 불변성을 보장하고 테스트하기 쉽다.\
#### 2. Setter 주입
Setter 메서드를 통해 의존성을 주입받는 방식. \
선택적 의존성이나 순환 참조가 필요한 경우에 사용된다.\
#### 3. Field 주입 (`@Autowired`)
필드에 직접 `@Autowired` 어노테이션을 사용하여 의존성을 주입받는 방식. \
가장 간편하지만, 테스트하기 어렵고, 순환 참조 문제를 일으킬 수 있어 **권장되지 않는다**.

## Spring Annotation
어노테이션은 스프링이 특정 기능을 수행하도록 지시하는 메타데이터의 역할을 한다. 사용할때는 코드에 `@`기호를 붙여 사용한다.\
예를 들어, `@Component` 어노테이션은 해당 클래스를 스프링 빈으로 등록하라는 의미를 가지며,`@Autowired`는 필요한 의존 객체를 자동으로 주입하라는 의미를 가진다.\
어노테이션을 활용하면 XML 설정 없이도 스프링의 다양한 기능을 쉽게 사용할 수 있다. 옛날에는 XML 파일에 매우 길게 작성하던 설정을 어노테이션을 통에 빠르고 쉽게 대체할 수 있다.\


### Spring Annotation 예시
스프링 부트에는 어노테이션이 엄청 많다. 예시에 적은 것보다 진짜 매우 많다. 필요할때마다 찾아봐야한다.

* `@Component` : 일반적인 빈을 등록한다. 여기서 빈은 스프링 컨테이너가 관리하는 객체를 의미한다.
* `@Bean` : 외부 라이브러리처럼 직접 제어할 수 없는 것들을 빈으로 등록할 때 메서드 위에 사용한다
* `@Service` : 비즈니스 로직을 처리하는 핵심 계층을 명시한다
* `@Repository` : 데이터 계층임을 명시하며, DB 관련 예외를 처리한다.
* `@Controller` : URL 요청을 처리하는 웹 계층을 명시한다 
* `@SpringBootApplication` : 스프링 부트 애플리케이션의 설정, 자동 설정, 컴포넌트 스캔을 활성화한다.
* `@RequesBody` : HTTP 요청의 본문에 담긴 JSON 데이터를 자바 객체로 변환한다
* `@ResponseBody` : 자바 객체를 HTTP 응답의 본문의 JSON 데이터로 변환하여 클라이언트에게 반환한다.
